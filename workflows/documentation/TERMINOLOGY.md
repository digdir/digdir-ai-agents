### Retningslinjer for begreper og oversettelser for teknisk språk

På norsk bruker vi **norske begreper der vi kan** og dokumenterer dem i **begrepskatalogen**. Unngå jargong - forklar tekniske begreper på enkelt norsk når mulig

#### Avklarte begreper og anbefalinger

| Unngå                                                             | Anbefalt begrep / praksis                                                               |
|-------------------------------------------------------------------|-----------------------------------------------------------------------------------------| 
| Inkonsistent bruk av *Tjeneste / App / Skjema*                    | Bruk begrepet som er definert for konteksten                                            |
| Inkonsistent bruk av *Instans / forekomst / eksemplar / utgave*   | Bruk begrepet som er definert for konteksten                                            |
| *Stateless app*                                                   | **Instansløst skjema**                                                                  |
| *Prefill*, *pre-fill*                                             | **Forhåndsutfylling**                                                                   |
| *Alert komponent*                                                 | **Varslingskomponent**                                                                  |
| *DatePicker*                                                      | **Datovelger**                                                                          |
| *abonnementsvalidering feiler*                                    | **feil ved validering av abonnement**                                                   |
|                                                                   | Skriv **offentlig gyldige sertifikater** framfor tekniske termer som "publicly trusted" |
|                                                                   | Bruk **TLS** framfor SSL i nye tekster (TLS er den moderne standarden)                  |
| Historiske utdaterte begrep som master/slave, whitelist/blacklist | Bruk funksjonelle begrep som primær/sekundær, tilgangsliste/blokkeringsliste etc.       |
